## 주제

Vector DB의 인덱싱 개념

### 출처

Indexing overview - Pinecone Docs

### 밀집 인덱스 Dense Indexes

- 책의 모든 내용을 꼼꼼하게 기록해둔 목차
- 아주 자세하게 정보를 찾아야할 때 유용

### 희소 인덱스 Sparse Indexes

- 책의 핵심 키워드만 뽑아서 정리해둔 목차
- 한 번에 처리할 수 있는 데이터 양이나 검색 속도에 제한이 있을 수 있음

### 네임 스페이스 Namespaces

- 데이터 분리 가능
- 관련된 데이터만 빠르게 훑어서 검색 속도가 빨라짐

### 벡터 임베딩

- 글, 그림 정보를 컴퓨터가 이해할 수 있는 숫자(벡터)로 변환하는 것
- 컴퓨터의 성능향상

#### 통합 임베딩

- 알아서 숫자로 바꿔주고 검색할 때도 알아서 숫자로 바꿔줌

#### 직접 벡터 사용

- 우리가 직접 번역해서 숫자로 만들어줌

### 데이터 가져오기

- 대량 가져오기 Importing
- 업서트 - 새로운 책 추가, 기존 책 내용 업데이트
