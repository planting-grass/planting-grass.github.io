Spring 애플리케이션에서 외부 설정값(properties, yaml)을 주입받을 때 가장 빈번하게 사용하는 @Value 어노테이션의 사용상 주의점과 @ConfigurationProperties와의 차이점을 정리했습니다.

1. @Value 사용 시 주요 주의사항
빈(Bean) 등록 여부와 주입 시점
@Value는 대상 클래스가 스프링 컨테이너에 의해 관리되는 Bean일 때만 동작합니다.

클래스에 @Component, @Service, @RestController 등의 어노테이션이 누락되어 컴포넌트 스캔 대상에서 제외되면, 해당 클래스는 단순한 일반 객체로 취급되어 값이 주입되지 않습니다.

주입은 빈의 생성 및 의존 관계 주입 단계에서 이루어지므로, 정적(static) 변수에는 기본적으로 사용할 수 없습니다.

적절한 주입 방식의 선택
일반적인 의존성 주입과 마찬가지로 세 가지 방식을 사용할 수 있으며, 각각의 특성을 고려해야 합니다.

필드 주입: 코드가 간결하지만 외부에서 변경하기 어렵고 테스트가 까다롭습니다.

생성자 주입: 가장 권장되는 방식입니다. 불변성(final)을 확보할 수 있고, 생성 시점에 모든 설정값이 유효한지 검증할 수 있습니다.

Setter 주입: 런타임에 값을 변경해야 할 때 유용하지만, 객체의 상태가 변할 수 있다는 위험이 있습니다.

설정 파일 경로와 우선순위
주입하려는 키값이 application.properties 또는 application.yaml에 정확히 정의되어 있는지 확인해야 합니다.

파일이 클래스패스(src/main/resources)에 위치해야 하며, 프로퍼티 파일이 여러 개일 경우 스프링의 우선순위 규칙(Command line arguments > Java System properties > OS environment variables > Application properties 순)을 숙지해야 의도치 않은 값 주입을 방지할 수 있습니다.
