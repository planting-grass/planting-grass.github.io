### 1일 1아티클
#### 요즘IT
###### Comet으로 보는 UX의 미래
**탐색 과정의 변화**
- 기존의 탐색 과정 : Home → Search → Filter → Compare → Checkout
- 의도 기반 탐색 과정 : User Intent → AI Processing → Result
  
**Comet의 핵심 기능**
1. 문맥 기반 AI 어시스턴트
  - 사이드바는 단순히 질문에 답하는 것에서 그치지 않고, 사용자가 보고 있는 페이지 능동적으로 분석 후 필요한 인사이트 선제안 (ex. 항공권 탐색 중이면, 여러 탭의 가격을 자동 비교해 표시)
2. 사용 의도까지 기억하는 브라우징
  - 브라우징 세션 전체에서 사용자의 목표 이해 및 유지 (ex. 한 탭에서 호텔 탐색하다가 다른 탭에서 식당 검색 시, 앞서 설정한 예산 등의 조건을 그대로 기억하고 반영)
3. 여러 사이트의 정보 통합
  - 사용자가 수십번 클릭해야 할 작업을 한번에 정리
  - 온라인 정보를 다루는 방식 자체의 새로운 패러다임 제시
  
**Comet과 기존 브라우저의 차이**
- 기존 주소창 대신 자연어 기반 인터페이스 중심 작동
- 브라우저는 하나의 세션 전체에서 맥락 유지
- 사용자가 지금 무엇을 묻는지 뿐만 아니라, **그 배경에 있는 의도**까지 함께 이해
  
**UX 팀이 생각해야 할 핵심 인사이트**
1. 사용자의 사고 전환은 생각보다 빠름
  - 의도 기반 인터페이스는 며칠만에 익숙해짐
  - AI가 60-70% 수준으로만 정상 작동해도 적응에 큰 어려움 X
2. AI의 진짜 강점
  - 여러 자료를 모아 비교 및 정리하는 작업의 성공률은 90% 수준
  - 단계별 절차를 따라가는 작업의 성공률은 30% 수준
3. 미래의 UX는 하나의 거대 시스템이 아닌, 분산된 AI 구조에 가까움
  - 역할이 다른 여러 AI 서비스가 유기적 협력 시 훨씬 자연스러움
  
**AI가 실패하는 방식**
- 괜찮은 실패 : AI가 한계 인정 (사용자의 의도를 충실히 이행하려 했으나 한계에 부딪혔을 때, 이유 설명 및 다음 선택지 제공) → 사용자 신뢰 유지
- 위험한 실패 : AI가 틀린 답을 정답인 척 제공 → 사용자 신뢰 하락
  
**Comet의 기타 기술적 과제**
- 성능 : 기본 브라우징 작업 시 Chrome보다 체감 속도 저하, 첫 페이지 로딩/여러 탭 동시 처리 시 차이 극심
- 보안 : AI가 기능이 아닌 인프라가 되는 순간, 기존 브라우저 모델로는 대응이 어려운 AI 브라우저 특유의 공격 경로 존재 → 다른 보안 패러다임 필요
  
**AI 중심 인터페이스에서 새롭게 살펴볼 지표**
- 의도 달성률 : 사용자가 원하는 목표가 실제로 이루어졌는가?
- 신뢰 위임 지수 : 사용자가 AI에게 얼마나 빠르게, 어떤 수준까지 작업을 맡기는가?
- 의도 지속률 : 플랫폼이나 화면이 바뀌어도 사용자의 의도가 얼마나 안정적으로 이어지는가?
  
**사용 가이드**
1. 자동화보다 정보 정리부터
  - AI 기능은 실행을 대신하는 것보다, 정보를 모으고 비교/정리하는 능력에 기반
2. 사용자 플로우보다 의도 플로우
  - 사용자가 '무엇을 이루고 싶어하는가'를 중점으로, 기존 네비게이션은 보완 장치로 설계
3. 신뢰는 작은 단계부터
  - 초기에는 위험이 낮은 작업부터 맡기며 신뢰를 서서히 구축
4. 실패 시 돌아가는 길 준비
  - AI가 비정상 처리 시, 사용자가 즉시 이전 상태로 돌아갈 명확한 경로 마련
