### 1일 1아티클
#### IT데일리
###### DBMS 동향
**`Oracle`**
- 전체 DBMS 랭킹 1위 유지
- 몇년간 완만한 감소세
- 데이터 저장 및 관리 환경이 **클라우드** 중심으로 변화 → 오라클 중심의 온프레미스 구조 유지 필요성 저하
- DBMS에 기업이 기대하는 것 : 대규모 시스템의 종속 → 클라우드 기반 확장성, 유연성
  
**`MySQL`**
- 전체 DBMS 랭킹 2위 유지
- 오라클보다 빠른 하락세
- 데이터 활용 방식의 변화
  - 단순 트랜잭션 처리 중심의 운영 → 실시간 처리, 복잡한 데이터 분석, 데이터레이크 운영, AI/ML 연계 등 수요 증가
  - 기본적 관계형 오픈소스 DB의 한계
- DBMS에 기업이 기대하는 것 : 대규모 시스템의 종속 → 클라우드 기반 확장성, 유연성
  
**`PostgreSQL`**
- 전체 DBMS 랭킹 4위 유지
- 꾸준한 상승세
- 무료 오픈소스 DB이면서, 엔터프라이즈 기능을 갖춘 기업형 DB
  - 복잡한 SQL 처리, 강력한 트랜잭션 관리, 멀티모델 데이터(JSON, 시계열, etc.) 지원, 다양한 확장성
  - 주요 클라우드 사업자들이 PostgreSQL 호환 DB 지원 → 오픈소스의 유연성 + 클라우드 운영 편의성 확보
  
**`Snowflake, Databricks`**
- 가파른 상승세
- RDBMS + 데이터 저장/처리/분석/AI 지원하는 데이터 플랫폼으로 확장
  - Snowflake : 클라우드 네이티브 기반 데이터웨어하우스, 데이터레이크, 데이터 공유 기능 통합 플랫폼 → 대규모 데이터 관리 용이
  - Databricks : 아파치 스파크 기반 데이터 처리, 엔지니어링, 머신러닝, AI 모델링 등 데이터 활용 전 과정 지원 플랫폼
    - `lakehouse` : 데이터레이크 유연성 + 웨어하우스 성능, AI/ML 중심 데이터 활용에 가장 적합
