###### AI Agent 구축 6단계
**구축 단계**
1. 측정 가능한 문제부터
  - 기술이 아닌, 문제의 중심에서 출발해야 함
2. 올바른 팀 구성
  - 초기 단계부터 IT, 자동화 엔지니어, 보안팀 등 구성원이 참여해야 함
  - 나중에 재작성하지 않고도 확장 가능한 구조 구축 가능
3. 데이터, 시스템, 채널의 체계적 매핑
  - `실행 시스템`, `지식 소스`, `상호작용 채널` 3영역으로 구분
  - `실행 시스템` : 실제 작업이 수행되는 영역 (ex. 옥타, 지라, 서비스나우, 액티브 디렉토리, etc.)
  - `지식 소스` : 에이전트가 참조하는 자료 (ex. 내부 문서, 해결된 티켓, 지식 베이스, 컨플루언스 페이지, etc.)
  - `상호작용 채널` : 사용자가 요청을 입력하는 통로 (ex. 슬랙, 팀즈, 맞춤형 포털, 이메일 기반 에이전트, etc.)
4. 프롬프트가 아닌, 도구 설계
  - 도구 : 에이전트가 수행 가능한 작업의 최소 단위
  - 도구의 조건 : 단일 기능 수행, 명확한 입력값 정의, 구조화된 결과 반환
5. 처음부터 거버넌스 설계
  - 프롬프트 인젝션 방어 기능
  - 민감 데이터 토큰화
  - 전 작업의 감사 로그
  - 고위험 작업 승인 워크플로우 적용
6. 사용자 업무 환경에 배포
  - 처음에는 단일 워크플로우로 시작
  - 신뢰도가 낮거나 SLA 목표가 위태로울 시, 단계적 확장
  - 효과 측정 지표 : 1단계 티켓 감소율, 평균 해결 시간, 에이전트 사용률 변화 추이
  
> AI Agent 활용 시 설계의 중요성 강조
