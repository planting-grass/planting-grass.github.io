### 1일 1아티클
#### 요즘IT
###### 리서치 도구
**문제 상황**
- 검색어 설정 잘못될 시, 조사 전체가 잘못됨 : 시작점은 검색어가 아닌, **내 상황**과 **조건** 이해
- 요즘 리서치의 핵심은 정보 수집보다, **탐색 설계**
  - 범위 : 어디까지 알아야 하는가?
  - 신뢰도 : 어느 정도의 근거가 필요한가?
  - 재사용성 : 무엇을 남길 것인가?
  
**1단계 : 전체 그림 파악**
- 모르는 영역에 진입 전, 짧고 빠르게 관련 지식을 훑는 도구 필요
- **조사 이전 방향 설정** : `지금 이 주제를 어떤 각도로 봐야 할까?` 를 정리하는 단계
- 결과물 : 다시 사용할 수 있는 **키워드 묶음**, **짧은 메모**
- 도구 : `Perplexity`, `oo.ai` → **맥락 브리핑**
  
**2단계 : 조사 초안 생성**
- 조사 질문 목록(What, Why, So What), 목차, 추정/확인된 사실을 분리한 표 작성
- 결과물 : **초안 문서**
- 도구 : `ChatGPT`, `Gemini`, `Claude` → **리서치 플랜 생성, 1차 초안 생성, 추가 질문 도출용** (요약용 아님!)
  
**3단계 : 지식으로 변환**
- 한 페이지 요약 수준으로, 재사용 가능한 정리 포맷을 만들기
- 정의, 현황, 핵심 수치, 쟁점, 결론, 다음 액션을 포함
- 결과물 : **높은 이해도**
- 도구 : `NotebookLM`, `라이너` → **개인용 지식 베이스**
  
**4단계 : 주장 검증**
- 최신성, 논쟁성, 균형, 직접 인용 가능성을 확인
- 결과물 : **신뢰도**, **논문**
- 도구 : `Scite`, `Elicit` → **인용 맥락 확인**
